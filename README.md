# Site-to-Site VPN на базе EcoRouter (без NAT)

В этом репозитории содержатся эталонные конфигурации двух маршрутизаторов **EcoRouter** для построения защищённого Site-to-Site VPN-туннеля между двумя локальными сетями организации.

## 📌 Особенности конфигурации

- **Протокол:** IKEv2 (автоматическое установление и переустановка ключей)
- **Режим:** туннельный (TUNNEL)
- **Шифрование IKE Phase 1:** AES-256-CBC, HMAC-SHA256, DH group 14 (modp2048)
- **Шифрование IKE Phase 2 (ESP):** AES-256-CBC, HMAC-SHA256
- **Аутентификация:** предварительный общий ключ `Site2SiteKey`
- **Трансляция адресов (NAT):** отсутствует для трафика между VPN-подсетями
- **Проброс NAT-Traversal:** включён (UDP 4500)

## 🧩 Топология и IP-адресация

| Устройство | WAN-интерфейс | LAN-интерфейс | Локальная подсеть |
|------------|---------------|---------------|--------------------|
| **ER1**    | 192.168.104.25/24 | 192.168.100.1/24 | 192.168.100.0/24 |
| **ER2**    | 192.168.104.26/24 | 192.168.200.1/24 | 192.168.200.0/24 |

Статические маршруты:
- ER1: `ip route 192.168.200.0/24 192.168.104.26`
- ER2: `ip route 192.168.100.0/24 192.168.104.25`

## 🚀 Быстрое развертывание

1. Скопируйте содержимое `ER1.txt` на маршрутизатор ER1, а `ER2.txt` – на ER2.
2. Убедитесь, что предварительный ключ `Site2SiteKey` совпадает на обоих устройствах.
3. Проверьте, что интерфейсы WAN имеют связность между собой (например, через коммутатор или прямую линковку).
4. Настройте статические маршруты до удалённых подсетей (уже присутствуют в конфигурациях).
5. Трафик между подсетями `192.168.100.0/24` и `192.168.200.0/24` будет автоматически шифроваться.

## 🔍 Проверка работоспособности

После применения конфигураций выполните на любом из маршрутизаторов:

```bash
show crypto-ipsec ike security-associations
show crypto-ipsec ike connections
show filter-map ipv4 FM-VPN
```

Ожидаемый результат:
- состояние **ESTABLISHED** для IKE SA,
- состояние **INSTALLED** для Child SA,
- наличие SPI-идентификаторов (in/out).

## 🧪 Тестирование связности

С клиента в подсети `192.168.100.0/24` (например, Windows с IP `192.168.100.10`) выполните:

```cmd
ping 192.168.200.10
```

С клиента в подсети `192.168.200.0/24` (например, Alt-Linux с IP `192.168.200.10`) выполните:

```bash
ping 192.168.100.10
```

Успешный обмен ICMP-пакетами подтверждает работу VPN-туннеля.
